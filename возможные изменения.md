## [Курсовая.ipynb] Нативный Qwen3.5-формат вместо Alpaca-шаблона

**Файл:** `Курсовая.ipynb`, ячейка 45 (подготовка датасета для LLM)

**Текущий код:**
```python
def format_prompt(row):
    return (
        f'### Instruction:\n{row["instruction"]}\n\n'
        f'### Response:\n{row["response"]}'
    )
```

**Лучше:**
```python
def format_prompt(row):
    messages = [
        {"role": "user",      "content": row["instruction"]},
        {"role": "assistant", "content": row["response"]},
    ]
    return tokenizer.apply_chat_template(messages, tokenize=False)
```

**Почему:** Qwen3.5-4B-Instruct обучался на своём нативном chat-формате (ChatML). При файнтюнинге на Alpaca-шаблоне модель видит незнакомый паттерн — это снижает эффективность обучения и ухудшает инференс, если там использовать `apply_chat_template`.

**Критичность:** Низкая — модель сойдётся в любом случае, для курсовой некритично.

## [webapp] Отсутствие обработки out-of-scope запросов

**Файл:** `webapp/inference.py` + классификатор интентов (модель в `models/`)

**Текущее поведение:**
Классификатор интентов обязан выбрать один из 27 обученных классов — класса "вне домена" нет.
При запросе вне домена (например, "напиши поэму") классификатор возвращает ближайший по TF-IDF класс
(наблюдалось: `review`), а LLM отвечает на запрос как обычно.

**Лучше:**
```python
# Добавить порог уверенности классификатора
proba = classifier.predict_proba([text])[0]
if proba.max() < 0.5:
    intent = "other"  # out-of-scope
else:
    intent = label_encoder.inverse_transform([proba.argmax()])[0]
```
Дополнительно — обучить классификатор с классом `other` на примерах вне домена,
и настроить LLM возвращать фиксированный отказ при `intent == "other"`.

**Почему:** Без порога система молча присваивает любому запросу один из 27 интентов,
что вводит пользователя в заблуждение и искажает метрики в продакшне.

**Критичность:** Низкая — для курсовой достаточно упомянуть как ограничение в Заключении.

## [models/] Хранение моделей на Hugging Face Hub вместо git

**Файл:** `webapp/inference.py`, `webapp/classifier.py`, `models/`

**Проблема:**
Репозиторий содержит тяжёлые веса модели (`models/qwen-qlora-support/`) и
`.joblib` классификаторы (`voting_clf.joblib` — 532 МБ, `clf_Random_Forest.joblib` — 263 МБ).
Коммитить это в git — плохая практика: история раздувается, клонирование становится невозможным.

**Почему Hugging Face Hub, а не Google Drive:**
- Создан именно для ML-моделей; бесплатно хранит файлы любого размера через Git LFS
- `transformers` нативно поддерживает HF repo ID в `from_pretrained()` — без скачивания вручную
- `hf_hub_download()` кеширует файлы в `~/.cache/huggingface/` — повторного скачивания нет
- Google Drive требует OAuth / сервисный аккаунт, `gdown` ненадёжен на больших файлах

**Что менять:**

*Qwen merged model* — `inference.py` уже готов: достаточно поменять переменную окружения:
```bash
# было (локальный путь)
MERGED_MODEL=/app/models/qwen-merged

# станет (HF repo ID)
MERGED_MODEL=username/qwen3.5-4b-qlora-support-merged
```
`AutoModelForCausalLM.from_pretrained(MERGED_MODEL, ...)` сам скачает модель при первом запуске.

*Sklearn-классификаторы* — добавить ленивую загрузку в `classifier.py`:
```python
from huggingface_hub import hf_hub_download

HF_REPO = os.getenv("HF_CLASSIFIERS_REPO", "username/support-classifiers")

def _get_model_path(filename: str) -> str:
    # Скачивает файл при первом обращении; при повторных — берёт из кеша
    return hf_hub_download(repo_id=HF_REPO, filename=filename)
```

**Что добавить в `.gitignore`:**
```
models/qwen-qlora-support/
models/*.joblib
```

**Критичность:** Низкая — для курсовой некритично, но без этого репозиторий невозможно
нормально клонировать. Стоит сделать перед публикацией финальной версии.
